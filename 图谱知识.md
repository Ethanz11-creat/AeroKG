## 项目背景与目标

### 项目背景

中国民用航空规章体系庞大复杂，包含数百部规章文件，传统的人工查阅和理解方式效率低下。本项目旨在利用大语言模型（LLM）技术，从非结构化的规章文本中自动抽取结构化知识，构建可查询、可推理的知识图谱。

### 核心目标

1. **自动化知识抽取**：将规章文本转化为结构化的知识图谱
2. **支持智能查询**：提供复杂的语义查询和关系推理能力
3. **合规性检查**：快速定位相关规章条款，辅助合规审查
4. **关联分析**：发现规章之间的隐含关系和冲突点

### 项目规模

- **数据规模**：16部民航规章，7,943个节点，7,927条关系
- **技术栈**：Python + LLM (DeepSeek-V3) + Neo4j Aura
- **开发周期**：从原型到生产级部署

#### 3. 分类过滤模块 (classifier.py)

**职责**：对文本块进行分类和过滤

**分类类别**：
- `DEFINITION`: 定义类（包含"是指"、"定义为"等）
- `RULE`: 规则类（包含"应当"、"不得"、"可以"等）
- `CONDITION`: 条件类（包含"在...情况下"、"除...外"等）
- `STRUCTURAL`: 结构性内容
- `NOISE`: 噪声数据（过滤掉）
- `REVIEW_REQUIRED`: 需要人工审核

**分类逻辑**：
```python
def classify(self, chunk: Dict, flags: Dict) -> Tuple[ChunkCategory, List]:
    text = chunk.get("text", "")
    
    # 优先级：噪声 > 定义 > 规则 > 条件 > 结构
    if flags.get("too_short"):
        return ChunkCategory.NOISE, []
    
    found_modalities = self._detect_modalities(text)
    found_definitions = self._detect_definitions(text)
    
    if found_definitions and not found_modalities:
        return ChunkCategory.DEFINITION, [...]
    
    if found_modalities:
        return ChunkCategory.RULE, [...]
    
    return ChunkCategory.STRUCTURAL, []
```

#### 4. LLM抽取模块 (llm_provider.py + extractor.py)

**职责**：使用大语言模型进行知识抽取

**核心设计理念**：
- **LLM-first**：所有实体、规则、条件、约束均由LLM抽取
- **规则辅助**：规则仅用于清洗、过滤、校验、归一化
- **异步并发**：支持批量并发抽取，提升效率

**LLM Provider设计**：
```python

    def __init__(self, api_key, base_url, model):
        self.api_key = api_key
        self.base_url = base_url  # 硅基流动 API
        self.model = model  # DeepSeek-V3
        self.temperature = 0.05  # 低温度保证稳定性
        self.max_tokens = 800
    
    def extract_chunk(self, chunk_id, text, metadata):
        # 构造Prompt
        system_prompt = """你是一个专业的中国民航法规知识图谱抽取专家。
        
【绝对规则】
1. 只能输出JSON格式
2. 不要输出解释、说明、Markdown
3. 所有字段必须存在
        
【抽取范围】
- 定义类：包含"是指""定义为"等
- 规则类：包含"应当/不得/可以/负责"
- 条件类：包含"在...情况下""除...外"

        user_prompt = f"""
文档信息:

doc_title: {metadata['doc_title']}

{text}

输出JSON:
        """
        
        # 调用API
```

3. **抽取范围**：明确定义抽取和不抽取的内容类型

**结果处理**：

```python
def _process_llm_result(self, chunk, llm_result):
    # 处理术语
    for term in llm_result.get("terms", []):
        term_node = TermNode(
            id=self._make_id("term", f"{term['name']}_{chunk_id}"),
            name=term['name'],
            normalized_name=normalizer.normalize_entity(term['name']),
            category=term['category'],
            confidence=term['confidence'],
            evidence_text=term['evidence_text'],
        )
        self.terms.append(term_node)
    
    # 处理规则
    for rule in llm_result.get("rules", []):
        rule_node = RuleNode(
            id=self._make_id("rule", f"{chunk_id}_{len(self.rules)}"),
            rule_type=rule['rule_type'],
            modality=rule['modality'],
            subject=rule['subject'],
            action=rule['action'],
            object=rule['object'],
            normalized_subject=normalizer.normalize_subject(rule['subject']),
            confidence=rule['confidence'],
        )
        self.rules.append(rule_node)
        
        # 创建关系
        self.edges.append(Edge(
            source_id=article_node.id,
            target_id=rule_node.id,
            relation_type="HAS_RULE",
        ))
```

#### 5. 标准化模块 (normalizer.py)

**职责**：对抽取结果进行标准化处理

**关键功能**：
- 实体名称标准化（全角转半角、去括号缩写）
- 别名解析（"民航局" → "中国民用航空局"）
- 术语归一化

**标准化规则**：
```python
class Normalizer:
    def normalize_entity(self, name: str) -> Tuple[str, str]:
        # 1. 全角转半角
        normalized = self._to_halfwidth(name)
        
        # 2. 去除括号缩写
        normalized = self._remove_parenthetical_abbreviation(normalized)
        
        # 3. 解析别名
        canonical = self._resolve_alias(normalized)
        
        return normalized, canonical
    
    def _resolve_alias(self, name: str) -> Optional[str]:
        ALIAS_MAP = {
            "中国民用航空局": ["民航局"],
            "通信导航监视": ["CNS"],
        }
        # 返回标准名称
        for full_name, aliases in ALIAS_MAP.items():
            if name in aliases or name == full_name:
                return full_name
        return None
```

#### 6. 验证模块 (validator.py)

**职责**：验证抽取结果的完整性和正确性

**验证维度**：
1. **节点验证**：ID唯一性、必填字段、置信度范围
2. **边验证**：关系类型合法性、节点存在性
3. **交叉验证**：术语与定义的对应关系、规则完整性

**验证逻辑**：
```python
class Validator:
    def validate_results(self, results: Dict) -> Dict:
        # 1. 节点验证
        for node_type, nodes in results.items():
            ids_seen = set()
            for node in nodes:
                # ID唯一性
                if node['id'] in ids_seen:
                    self.warnings.append(f"Duplicate ID: {node['id']}")
                
                # 置信度范围
                if not (0 <= node['confidence'] <= 1):
                    self.warnings.append(f"Invalid confidence: {node['confidence']}")
        
        # 2. 边验证
        all_node_ids = {n['id'] for nodes in results.values() for n in nodes}
        for edge in results['edges']:
            if edge['source_id'] not in all_node_ids:
                self.warnings.append(f"Source node not found: {edge['source_id']}")
        
        # 3. 交叉验证
        term_names = {t['name'] for t in results['terms']}
        def_term_names = {d['term_name'] for d in results['definitions']}
        orphan_defs = def_term_names - term_names
        if orphan_defs:
            self.warnings.append(f"Orphan definitions: {len(orphan_defs)}")
        
        return {
            "is_valid": len(self.errors) == 0,
            "warnings": self.warnings,
        }
```

#### Step 1: 文本清洗

**目标**：提高文本质量，减少噪声干扰

**清洗策略**：

1. **格式规范化**
   ```python

   text = text.translate(FULLWIDTH_MAP)
   
   # 统一空格
   text = re.sub(r'\s+', ' ', text)

   text = re.sub(r'([，。；：])\1+', r'\1', text)
   ```

2. **噪声过滤**
   ```python
   # 页码
   text = re.sub(r'-\s*\d+\s*-', '', text)
   
   # 表格/图片标记
   if re.search(r'(表\d+|图\d+|附件)', text):
       flags['is_table_or_figure'] = True
   
   # 附录块
   if re.search(r'^(附表|申请表|备案表)', text):
       flags['is_appendix_block'] = True
   ```

3. **可疑内容标记**
   ```python
   # 跨句串接检测
   if re.search(r'。[^，。；]+应当', text):
       flags['suspicious_article_match'] = True
   
   # 长度异常
   if len(text) < 15:
       flags['too_short'] = True
   elif len(text) > 3000:
       flags['review_required'] = True
   ```

**目标**：识别文本类型，过滤噪声，提高抽取效率

#### Q2: 为什么选择知识图谱而不是传统数据库？

**回答要点**：

选择知识图谱主要基于以下考虑：

**1. 数据特性匹配**
- 规章数据天然具有层次结构和关联关系
- Document → 章节 → 条款 → 规则，形成清晰的层级
- 规则之间存在大量引用、关联关系

**2. 查询需求匹配**
- 需要支持复杂的关联查询（如"查找所有引用某法规的规则"）
- 需要支持多跳查询（如"查找某主体的所有职责"）
- 需要支持路径查询（如"规章A和规章B之间的关联路径"）

**3. 推理能力**
- 支持基于规则的推理（如规则冲突检测）
- 支持语义推理（如主体职责推理）
